19 / 36 講義 · PRAC4-3
Validation · 工具有沒有脾氣

同樣一份空白表單，兩種對待方式

壞工具 · 沒驗證
「畢業於 XX 大學，曾於 XX 公司擔任 XX 職務」

空白也照生、沒料也照寫——產出空洞八股，使用者拿到還要刪一遍，根本不能用。

好工具 · 有驗證
「請補充：學歷 / 工作內容（少於 10 字） / 技能（至少 2 項）」

缺什麼明確列出、紅框標出哪一欄，不執行生成。逼使用者補齊，產出才會真有料。

設計準則：先想清楚「哪些欄位是生成的最低必要條件？空白時工具該做什麼？」——把這個判斷寫進 Prompt，工具就有這個脾氣。

Design Detail · 工具的兩個小脾氣
22 / 36 講義 · PRAC4-3
Design Detail · 細節決定工具的人格

兩個藏在規格裡的小脾氣

脾氣 1 · 重新生成 = 覆蓋
不疊加，每次重來一份

使用者改了素材重按生成 → 直接覆蓋舊輸出，不要往下疊加。畫面亂了使用者就不會用第三次。

脾氣 2 · 輸出是草稿
下方加事實核對提醒

「請核對段落中的公司名稱、職稱、年資是否正確」——AI 會把素材重組得很順，但細節對不對是你自己要負責。

本練習的核心收穫：把這兩個脾氣寫進 Prompt 是「從生成器升級到工具」的分水嶺——很多人寫了 Prompt，但從沒想過工具該對使用者「有要求」。

Prompt Template · 直接照抄、上線可用

必寫進 Prompt 的 5 區塊
· 輸入欄位（必填 / 選填明列）
· 版本選擇（4 種 + 字數）
· 語氣風格（4 種單選）

設計規格區塊
· 風格：無印良品（背景 #f5f3ee / 卡片白）
· 邊框：#d8d4cb · 強調：霧藍 #6b7fa3
· 字體：Shippori Mincho + Noto Sans TC
· 圓角 6px · 無漸層 · 無陰影
· 不依賴外部 library
關鍵句：「驗證未通過時顯示紅色提示區塊，列出所有缺少的項目，不執行生成」——少這一句，AI 就會在素材不全時硬擠一份段落，工具就毀了。

30 / 36 講義 · PRAC4-4
Debug Drill · 從 Part 1 一路接到這裡

同一個壞 Prompt，差別在三件事

原始（壞掉的）
幫我設計一個 AI 工具，可以幫公司客服部門節省時間。
要好用、穩定，讓同事願意用。

「節省時間」太籠統 / 「好用穩定」是 UX 原則不是規格 / 沒講清楚現有流程。

修正版三大關鍵改動
① 「節省時間」→ 具體說明哪類重複工作佔多少比例（60% 是退換貨等）
② 「好用穩定」→ 明確功能邊界（初篩 + 草稿、不自動發送）
③ 加技術限制（Apps Script 或 Make.com）+ 要求輸出架構圖 + 時間估算
本練習真正在學的：不是「修這一條」，是「下次自己寫 Prompt 時的自我審稿」——「我這句話 AI 能不能驗收？」能就過、不能就改。
